🚀 小程序后台服务器 Demo

基于 Node.js + Express + MySQL + Redis 的分层架构脚手架

Node.js Express 4 MySQL · mysql2 Redis · ioredis JWT

项目简介

这是一个面向微信小程序后端的脚手架项目,采用清晰的分层架构, 内置一套完整的「微信登录 → JWT 鉴权 → 用户资料读写 + Redis 缓存」示例, 开箱即用,便于在此基础上扩展业务。

routes → controllers → services → repositories(MySQL) / loaders(Redis)

技术栈

分类选型说明
运行环境Node.js 18+使用原生 fetch,无需额外 HTTP 库
Web 框架Express 4轻量、生态成熟、上手快
数据库MySQL + mysql2Promise 连接池,支持命名占位符
缓存Redis + ioredis会话存储 + 资料缓存
鉴权jsonwebtokenJWT + Redis 会话双重校验

目录结构

miniapp-server/
├── src/
│   ├── server.js              # 启动入口:初始化基础设施 + 监听 + 优雅关闭
│   ├── app.js                 # 组装 Express(中间件 / 路由 / 错误处理)
│   ├── config/                # 配置层:统一读取环境变量
│   ├── loaders/               # 基础设施层:MySQL 连接池 + Redis 客户端(单例)
│   ├── routes/                # 路由层:URL 与控制器映射
│   ├── controllers/           # 控制器层:解析请求 / 返回响应
│   ├── services/              # 业务逻辑层:编排数据访问与缓存
│   ├── repositories/          # 数据访问层(DAO):只写 SQL
│   ├── middlewares/           # 中间件:鉴权 / 错误处理 / async 包装
│   └── utils/                 # 工具:日志 / 统一响应 / 业务异常
├── sql/
│   └── init.sql               # 建库建表脚本
├── doc/
│   └── index.html             # 项目文档(本页)
├── .env.example               # 环境变量样例
└── package.json

每一层职责单一,上层依赖下层,便于单元测试与实现替换。

核心设计

连接池单例

MySQL/Redis 在进程启动时初始化一次,全局复用,避免频繁建连。

Cache-Aside 缓存

读资料先查 Redis,未命中查 MySQL 并回填;写资料后删除缓存。

Redis 会话

登录时把 token 存入 session:{userId},鉴权中间件比对,支持服务端踢登录。

统一响应 / 错误

{ code, message, data } 格式 + 全局错误中间件 + ApiError 业务异常。

优雅关闭

收到信号先停收请求,再关闭连接池/客户端,避免请求中断。

本地 Mock 登录

未配微信 appid 时使用 mock openid,无微信账号也能跑通全流程。

API 接口

方法路径鉴权说明
GET/api/health否健康检查
POST/api/users/login否小程序登录,返回 token
GET/api/users/profile是获取用户资料(Redis 缓存)
PUT/api/users/profile是更新用户资料(更新后清缓存)
POST/api/users/logout是退出登录(删除 Redis 会话)

快速开始

# 1. 安装依赖
npm install

# 2. 准备配置
cp .env.example .env       # 按需修改 MySQL / Redis / 微信 / JWT 配置

# 3. 初始化数据库
mysql -u root -p < sql/init.sql

# 4. 启动(需本地已启动 Redis)
npm run dev                # 开发模式(nodemon 热重载)
# 或
npm start

登录并调用受保护接口

# 登录(前端用 wx.login 拿到的 code,这里用任意字符串模拟)
curl -X POST http://localhost:3000/api/users/login \
  -H 'Content-Type: application/json' \
  -d '{"code":"test_code_123"}'

# 用返回的 token 访问资料
curl http://localhost:3000/api/users/profile \
  -H 'Authorization: Bearer <token>'
